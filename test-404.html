How to Test 404 Functionality

Click the links below to test different 404 scenarios. These will help verify that the 404 page is properly configured and working.

Test Links (These should trigger 404 errors):

Non-existent Page Missing Folder Old PHP Page Random Text

Auto-Redirect Tests (These should redirect automatically):

/home → homepage.html /portfolio → portfolio.html /about → studio.html /contact → contact.html

Expected Behavior:

  • 404 Errors: Should show the custom 404 page with search functionality
  • Auto-Redirects: Should automatically redirect to the correct page
  • Search: Should allow users to search for available pages
  • Navigation: Should provide easy ways to get back to main content

Server Configuration:

Make sure your web server is configured to serve the 404.html page for 404 errors. The .htaccess file should include:

ErrorDocument 404 /404.html
